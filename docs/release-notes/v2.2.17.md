# v2.2.17

v2.2.17 是 v2.2.16 之后的维护版本，重点修复 SMTP SSL、备份包下载、脚本文件树隐藏文件显示、依赖状态显示、环境变量排序体验，并补齐 Python 3.10 / 3.11 / 3.12 多版本运行与依赖隔离能力。

## 新增

- **Python 多版本运行环境**：面板新增 Python 3.10、3.11、3.12 版本选择能力。依赖管理页面可以切换查看不同 Python 版本的依赖，也可以把当前版本设为默认版本。
- **任务级 Python 版本选择**：任务表单新增 Python 版本选项，Python 任务运行、缺依赖自动安装、任务导入导出都会保留并使用对应版本。
- **Python 依赖隔离**：Python 3.10、3.11、3.12 使用独立托管依赖环境。Python 3.12 继续兼容原有 `deps/python/venv`，避免升级后已安装依赖丢失；3.10 / 3.11 分别使用独立目录。
- **Python 依赖同步安装**：在依赖管理里安装 Python 依赖时，会同时为 Python 3.10、3.11、3.12 创建安装任务，后续切换任务 Python 版本无需再次手动安装同名依赖。
- **终端 IP 白名单恢复命令**：`ddp` 新增 `ip-whitelist` 命令，支持 `list`、`add`、`delete`、`clear`、`set`。当用户把 IP 白名单设置错误导致无法进入面板时，可在终端执行 `ddp ip-whitelist clear` 清空白名单并恢复访问。

## 修复

- **SMTP 邮件通知支持 SSL**：邮件通知渠道新增 `SSL 连接` 配置，可选择自动、启用 SSL 或关闭 SSL。后端发送邮件时支持 465 端口常用的隐式 SSL 连接；未显式配置时，端口为 `465` 会自动启用 SSL，其它端口继续沿用普通 SMTP / STARTTLS 发送逻辑。青龙配置导入的 SMTP 465 邮件渠道会自动标记为启用 SSL。
- **备份数据包下载失败**：修复数据备份与恢复页面点击“下载”后备份包无法正常保存的问题。备份下载接口新增查询参数下载入口，避开部分代理或路由对 `.tgz` / `.tar.gz` 文件名路径参数的兼容问题；同时补充文件存在性、文件类型与路径安全校验，前端下载失败时会展示后端返回的具体错误原因。
- **脚本文件树显示不完整**：修复脚本文件页面不显示 `.env`、`.config`、`.xxx` 等点开头文件或文件夹的问题。脚本文件树现在会展示脚本目录下的完整文件结构，同时继续过滤 `node_modules`，并新增过滤 Python 调试缓存目录 `__pycache__`。
- **依赖恢复与启动校验按 Python 版本处理**：备份恢复、重启后的依赖校验、依赖清单导出现在会区分 Python 版本，避免不同 Python 版本之间误判已安装或漏装。
- **Node.js 依赖状态显示异常**：修复重启或更新面板后，`chalk@4.1.2`、`http-proxy-agent@7.0.0` 等带版本号安装记录可能被误判为失败的问题。启动校验现在会按实际 npm 包目录识别依赖是否已安装。
- **环境变量全部分页拖拽失效**：修复环境变量页面切换到“全部”后无法拖拽排序的问题，全部列表与 20 / 50 / 100 分页保持一致的排序体验。
- **系统更新加速源配置**：网络代理设置中移除单一“推荐镜像”按钮，改为“配置”弹窗展示可选 Docker 更新镜像源；新增“二进制更新加速源”配置，二进制部署后台更新下载 GitHub Release 包时可通过加速源转发下载。
- **Docker 更新提示优化**：调整 Docker 部署未挂载 Docker Socket 时的检查更新提示，改为优先说明推荐使用 Watchtower 托管自动更新；需要立即手动更新时再提示宿主机执行 `docker compose pull && docker compose up -d`。
- **表格复选框对勾显示异常**：修复任务、订阅、依赖、日志等表格批量勾选时，复选框内对勾错位或显示不完整的问题。
- **全局前置脚本参数传递失败**：修复 `task_before.sh`、`task_after.sh`、`extra.sh` 以及任务内置前置/后置脚本无法通过 `$1`、`$2` 获取当前任务脚本参数的问题。全局前置脚本现在会与目标脚本收到同一份参数，便于统一代理、账号或运行前环境初始化。
- **定时任务日志显示不全**：修复任务日志弹窗在实时流已收到部分内容后，任务结束时不再回读最终落盘日志，导致显示内容少于调试运行日志的问题；同时加固 SSE 多行日志传输格式，避免多行输出在实时日志中丢失。
- **登录验证码触发策略**：极验验证码开启并配置完整后，登录不再等待同一 IP / 用户名连续失败 3 次才触发，而是每次登录都先要求完成滑块验证，提升登录入口安全性。
- **任务超时设置调整**：移除系统设置中的“全局默认超时（秒）”，任务执行只使用单个任务自己的超时配置；新建任务、导入任务、订阅自动创建任务默认超时为 `0`，表示永不自动超时。

## 优化

- **自定义背景图片大小限制**：日志背景图片上传上限从 2MB 提升到 10MB，便于使用更清晰的自定义图片。
- **订阅设置文案精简**：精简订阅管理中“覆盖拉取”的辅助说明，减少设置弹窗里的说明文字干扰。
- **脚本编辑器体验优化**：脚本管理页面打开脚本后直接使用 Monaco Editor 进行只读查看，默认提供代码高亮、行号和编辑器交互；进入页面后会预加载 Monaco，减少首次编辑等待。
- **更新网络提示去品牌化**：Docker 更新网络异常提示不再内置固定镜像域名示例，避免对某个镜像源产生默认推荐含义。
- **接口文档标题精简**：接口文档页面标题移除装饰图标，仅保留“开发接口文档”文字。
- **任务前后置脚本说明优化**：任务表单中的“钩子脚本”改为更直观的“前后置脚本”，并补充前置/后置执行时机与 `$1`、`$2` 参数读取说明。
- **代理地址说明优化**：网络代理设置中的“代理地址”新增说明按钮，解释该配置的作用、适用场景、填写格式以及 Docker 容器内填写 127.0.0.1 的注意事项。
- **登录验证码体验优化**：登录页不再显示内嵌验证码提示面板；点击登录后直接拉起极验滑块弹窗，滑块验证成功后自动继续提交登录，无需再次点击登录按钮。
- **部署文档精简**：README 移除冗长的生产版 Watchtower 扩展说明，Docker 示例默认改回官方镜像名，镜像加速源改为用户按需自行配置，并补充可到 `status.anye.xyz` 查看可用镜像源状态。

## 升级说明

- 旧版本已有 Python 依赖和任务会按 Python 3.12 处理，继续使用原来的托管环境目录。
- 如需运行 Python 3.10 或 3.11 任务，请确保服务器系统中已安装对应解释器；面板会在首次使用时为该版本创建独立虚拟环境。
