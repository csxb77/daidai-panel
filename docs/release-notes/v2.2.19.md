# v2.2.19

v2.2.19 是 v2.2.18 之后的维护版本。  
这次更新主要围绕两条主线展开：一条继续处理任务运行、依赖恢复、订阅拉库和 Python 运行环境的稳定性问题；另一条集中收口面板整体 UI、Loading、交互动效、暗色模式和移动端体验，让面板在“更稳”的同时，也更统一、更顺眼。

## 新增

- **统一 Loading 视觉体系**：面板新增统一的 Loading 展示组件，支持 `infinity`、`spinner`、`dots` 三种动态样式，并可按尺寸、色调和是否堆叠文案切换展示。登录页、依赖管理、订阅管理、按钮加载态、Monaco 编辑器加载态和局部空白区加载态会逐步统一到这一套视觉语言。
- **核心加载态升级为 SVG 动画**：在统一 Loading 组件的基础上，进一步把核心加载态升级为内联 SVG 动画版本。全局 `v-loading` 遮罩、按钮 loading、局部页面 loading 会优先使用更细腻的 SVG 动效，整体观感更完整，也更符合当前面板的新视觉风格。

## 修复

- **脚本缺依赖自动安装偶发失效**：修复任务执行器构建脚本运行环境时把 `workDir` 错传成脚本根目录的问题。现在任务运行、缺依赖检测和自动安装重试会优先使用脚本实际所在目录，减少“明明检测到缺依赖，但自动安装没有真正生效”的情况。
- **Node.js 依赖安装报 `EJSONPARSE`**：修复 `data/deps/nodejs/package.json` 损坏后，Node.js 依赖安装会被直接卡死的问题。现在安装、重装和自动补依赖前会先校验 `package.json`，发现 JSON 损坏会自动备份为 `package.json.broken-*`，再根据现有 `node_modules` 重建可用的最小清单，避免 npm 因历史坏文件直接退出。
- **订阅 GitHub 仓库仍会把无关文件完整拉下**：修复订阅管理里已经填写“指定子目录”或“白名单”时，首次 `git clone` 仍会先把仓库根目录和无关文件检出到 `scripts` 的问题。现在有路径过滤时会先进入 sparse checkout，再按规则检出；子目录会改用非 cone 模式，白名单也会参与真实检出范围，减少大仓库无关文件落盘。
- **Debian 容器依赖长期卡在“等待重启安装”**：修复 Debian 版在服务重启后恢复 Linux 依赖时，仍走旧的裸 `apt-get update && apt-get install ...` 逻辑，导致与当前依赖管理页的安装策略分叉、状态长期不收敛的问题。现在 Linux 依赖的正常安装和“重启后自动重新安装”会统一复用同一套包管理器策略，Debian / Alpine 的行为保持一致。
- **坏掉的 Python 虚拟环境备份持续堆积**：修复 `deps/python/3.12.broken-时间戳` 这类坏 venv 备份目录长期残留的问题。面板启动时会自动清理旧的 `.broken-*` 备份目录，避免 `deps/python` 目录持续膨胀。
- **脚本管理移动端工作区显示不稳定**：修复脚本管理页在移动端下脚本列表、工作区和编辑器切换不够稳定的问题。现在移动端会更明确地区分目录区、内容区和编辑区的切换状态，减少出现空白区、层级混乱或操作区域突然消失的情况。
- **部分页面仍残留旧的紫色主题元素**：修复仪表盘、开发接口、个人中心、全局增强样式等区域仍残留紫色高光、紫色阴影或蓝紫渐变的问题。本轮已统一收口为更克制的蓝 / 青 / 绿体系，避免页面局部视觉跑偏。
- **后台与暗色模式局部样式不一致**：修复日志页、脚本侧栏、脚本树节点、设置页概览卡片以及后台若干卡片在暗色模式下边框、背景、分割线和交互高亮不统一的问题，减少深色环境下“有的发灰、有的发紫、有的发白”的割裂感。

## 优化

- **面板启动更轻**：启动时不再默认预热 `3.10 / 3.11 / 3.12` 三个 Python 版本的托管环境。现在会先做历史目录迁移和坏环境清理，真正需要运行脚本、安装依赖或使用对应版本时再按需初始化，降低空闲时的启动负担和文件缓存压力。
- **订阅拉库更省空间**：对带子目录或白名单的 Git 订阅使用 `--no-checkout` + sparse checkout，并在远端支持时启用 `--filter=blob:none`，尽量减少首次拉取时无关 blob 下载和工作区文件落盘。
- **Node.js 依赖安装更稳定**：Node.js 依赖的安装、卸载、强制卸载、备份恢复重装和脚本自动补依赖现在会串行执行，避免多个 npm 进程同时改同一个 `package.json / package-lock.json`，降低清单文件被并发写坏的概率。
- **Python 运行环境目录更收敛**：在保留多版本能力的前提下，减少“默认全量预热 + 保留坏环境备份”带来的额外磁盘占用，降低 `deps/python` 目录膨胀速度。
- **Linux 依赖安装链路统一**：Linux 依赖的安装、重装、重启后恢复现在共享同一套底层包管理器命令构建逻辑，后续 Alpine、Debian 和其它发行版的行为更容易保持一致，也更方便继续维护镜像源、索引刷新和锁冲突处理。
- **Loading 观感进一步统一**：登录页、依赖管理、订阅管理、编辑器、按钮、局部卡片和全局遮罩的 Loading 风格做了统一收口。相比之前各页自己写一套小圈、小点、小条，现在视觉更一致，等待时不再像“拼出来的页面”。
- **核心页面视觉整体升级**：登录页角色区、依赖管理页、通知页、开发接口页、仪表盘、个人中心等核心页面做了多轮细节精修，包括标题区、卡片层次、筛选工具栏、空状态、统计区和操作区的颜色与留白统一，整体更偏干净的浅色蓝青风格。
- **页面交互动效更顺滑**：主布局页切换、卡片 hover、按钮按压、抽屉 / 弹窗出现、日志区切换、编辑器侧栏展开收起等交互补齐了更克制的过渡效果，减少生硬闪现和组件各自动的割裂感。
- **登录后高频页面继续打磨**：仪表盘、日志页、脚本管理页、脚本编辑器、脚本树节点、脚本侧边栏等登录后高频页面做了连续多轮动效和层次收口，让常用操作区更有“面板感”，而不是只靠 Element 默认样式堆叠。
- **暗色模式体验继续收口**：主布局、日志页、脚本管理、设置页概览卡片等暗色模式重点区域重新调整了背景层次、边框透明度、阴影、选中态与 hover 态，减少暗色下发脏、发糊、发闷的问题。
- **管理后台观感继续统一**：用户管理、开发接口、依赖管理、系统设置等后台页的顶部工具条、统计区、卡片操作区、标签页、表格和弹窗观感进一步统一，后台不再是一页一个风格。
- **移动端体验继续补齐**：仪表盘、日志、脚本管理以及后台若干页面针对小屏设备继续做了布局收口，减少按钮挤压、工具栏换行突兀和局部区域难点按的问题。

## 升级说明

- 升级到 v2.2.19 后，首次启动会自动清理 `deps/python` 目录下历史遗留的 `.broken-*` 备份环境；如果你仍需要保留这些目录做排查，请先手动备份。
- 从 v2.2.19 开始，面板启动阶段不会再主动预热三个 Python 版本的托管环境；如果某个版本之前从未用过，首次运行对应 Python 任务或首次安装该版本依赖时，仍会按需初始化。
- Debian 部署如果之前有 Linux 依赖长时间停在“等待重启安装”或“安装中”，升级并重启容器后，建议进入依赖管理页观察这些依赖的状态是否已自动收敛；若旧状态仍停留不动，可手动点一次“重装”触发新逻辑。
- 如果某个订阅之前已经把整个仓库完整拉进了脚本目录，升级后建议重新拉取一次该订阅，让工作区按新的子目录 / 白名单规则重新收敛；清空子目录和白名单后再次拉取，则会自动恢复完整仓库检出。
- 如果之前 Node.js 依赖安装曾出现 `EJSONPARSE` 或 `Invalid package.json`，升级后直接重新安装或重装对应依赖即可；原损坏文件会保留为 `data/deps/nodejs/package.json.broken-*` 方便排查。
- 本次版本同时包含一轮前端静态资源更新。如果升级后二级页面仍看到旧的 Loading、旧的配色或旧的动效，建议手动强制刷新浏览器缓存后再看效果。
- 本次更新不涉及数据库结构变更，主要影响任务运行环境、Node.js 依赖清单修复、订阅 Git 拉取工作区、Python 托管目录清理、Linux 依赖恢复逻辑，以及前端 Loading、UI 和动效展示。
