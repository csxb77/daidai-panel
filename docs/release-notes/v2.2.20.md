# v2.2.20
<!-- release-title: 日志交互与模块运行时修复 -->

## 版本概览

`v2.2.20` 是 `v2.2.19` 之后的一次针对高频实际问题的修复版本。  
本次更新重点不在功能扩张，而在于把三个已经影响真实使用的问题直接收口：

- **日志体验修复**：任务进度条、下载器、CLI 状态条这类依赖回车覆盖刷新的输出，现在会在面板里按单行动态刷新显示，不再疯狂刷屏。
- **编辑器可用性修复**：修复部分用户升级到 `v2.2.19` 后 Monaco 编辑器加载失败的问题，恢复本地静态资源完整性，避免只能依赖 CDN 才能打开编辑器。
- **模块版 Python 运行时修复**：修复 Magisk / KernelSU / APatch 模块版在单 Python 环境下被错误判定为 `Python 3.12 不可用`，导致历史 Python 任务整体失效的问题。

## 新增

- **补充日志流回归保护**：为任务实时日志 SSE 输出新增回归测试，明确约束后端必须保留裸 `\r` 的终端覆盖刷新语义，避免后续再把进度条输出重新打散成多行。
- **补充模块版 Python 版本回归测试**：新增 Magisk 模块运行时相关测试，确保模块版默认 Python 版本会跟随容器里真实 `python3` 小版本，而不是继续被固定成 `3.12`。

## 修复

- **任务日志中的进度条会把日志刷满整页**：修复任务实时日志、执行日志详情、日志文件预览这三条链路对裸回车 `\r` 的处理错误。现在进度条、下载状态、终端动态覆盖输出会在同一行内刷新，普通日志不会再被每秒一条的“假换行”淹没。
- **编辑器加载失败，提示“请检查网络或稍后重试”**：修复 `v2.2.19` 中 Monaco 本地静态资源被裁剪过头的问题。之前 `loader.js` 仍在，但 `editor/`、`language/`、`assets/`、`basic-languages/` 等运行时必需目录缺失，前端会误判本地可用，真正初始化时直接失败。现在已恢复本地完整资源，编辑器不再只靠 CDN 才能打开。
- **Monaco 本地资源可用性误判**：修复前端原先只检查 `loader.js` 的探测逻辑。现在会同时检查 `editor/editor.main.js`、`editor/editor.main.css` 以及关键语言入口文件，避免“资源残缺但仍被当成可用”的情况。
- **模块版 Python 脚本整体报“Python 3.12 不可用”**：修复 Magisk / KernelSU / APatch 模块版在单 Python 容器环境下与多版本 Python 逻辑不匹配的问题。现在模块版会优先按容器里真实 `python3` 小版本对齐任务运行时，不再因为历史默认值仍是 `3.12` 而让 Python 任务整体失效。
- **模块容器会把托管 Python venv 错建到 `deps/python/3.12`**：修复模块启动脚本里对 Python venv 目录的硬编码。现在启动时会先探测容器里真实 `python3` 小版本，再把托管环境建到对应目录，避免“目录名是 3.12，实际却是 3.11 venv”这种错位状态。

## 优化

- **日志显示语义更接近真实终端**：实时日志、历史日志和日志文件详情页现在统一按终端语义解释 `\n`、`\r\n`、`\r` 三类边界，交互一致性更高。
- **编辑器本地资源策略更稳**：Monaco 资源复制脚本不再依赖带 hash 的文件名白名单去“猜”哪些文件能删，减少后续升级 Monaco 或调整构建时再次删错运行时文件的风险。
- **模块版运行时默认行为更符合实际环境**：模块版默认 Python 版本现在会优先和当前容器里真实 `python3` 小版本保持一致，减少历史任务、默认配置和实际解释器之间的偏差。

## 升级说明

- **所有部署形态**：本次不涉及数据库结构变更，主要修复日志展示、编辑器加载与模块版 Python 运行时行为。
- **实时日志 / 执行日志用户**：升级后再次查看带进度条的脚本执行日志时，面板会按单行覆盖刷新显示。如果浏览器仍看到旧表现，建议强制刷新前端缓存后再观察。
- **脚本编辑器用户**：本次包含 Monaco 本地静态资源修复。升级后如果之前遇到“编辑器加载失败”，请重新打开脚本编辑页验证；若浏览器缓存较重，建议手动刷新缓存。
- **Magisk / KernelSU / APatch 模块用户**：本次版本默认仍按**单 Python 实际运行时**工作，不会自动安装三套 `3.10 / 3.11 / 3.12` 解释器。升级后模块版会优先对齐容器里真实 `python3` 小版本，历史 Python 任务不应再因为默认值残留 `3.12` 而整体报错。
- **模块版排查建议**：如果升级后仍怀疑 Python 环境异常，可先在模块卡片里点「运行 / Action」查看容器运行时输出，确认当前检测到的 `python3` 路径和实际版本，再决定是否需要进一步排查容器环境。
