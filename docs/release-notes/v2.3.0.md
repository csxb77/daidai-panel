# v2.3.0
<!-- release-title: Web 日志与反代访问修复 -->

## 版本概览

`v2.3.0` 是 `v2.2.20` 之后的一次稳定性修复版本，重点解决两个用户反馈的高频实际问题：

- Web 面板中进度条类日志仍然刷屏，和终端单行刷新表现不一致。
- 群晖、飞牛等 NAS 容器通过 Nginx Proxy Manager / 第三方反代公网访问时，登录接口被误判跨域并返回 403。

本次不涉及数据库结构变更，也不需要用户手动迁移数据。

## 修复

- **Web 面板进度条日志仍刷屏**：修复任务实时日志在 SSE 解析时丢失裸 `\r` 控制符的问题。现在脚本里的进度条、下载状态等终端覆盖刷新输出会继续在同一行更新，不再因为每个流式分片都被当成新行而刷屏。
- **执行日志详情和日志文件预览表现不一致**：统一任务实时日志、执行日志详情、日志文件预览三处对裸 `\r`、`\n`、`\r\n` 的处理语义，避免任务弹窗正常、历史详情或日志文件仍刷屏的问题。
- **NAS + Nginx Proxy Manager 反代登录提示跨域**：修复群晖、飞牛等 NAS 容器通过公网域名和非标准端口反代访问时，反代只传域名却丢失外部端口，导致 `Origin=https://域名:端口` 被误判为跨域并拦截登录的问题。
- **反代端口安全校验边界**：当反代明确传入 `X-Forwarded-Port` 时，后端仍会严格校验该端口必须与浏览器 `Origin` 端口一致；不同域名或明确端口冲突仍会被拒绝，避免为了兼容反代而放大 CORS 风险。

## 优化

- **日志渲染更接近真实终端**：Web 日志组件现在按终端语义区分真实换行和单行覆盖刷新，普通多行日志仍保持原样展示。
- **CORS 拒绝日志更利于排查反代问题**：CORS 拒绝日志补充 `X-Forwarded-Port` 与 `X-Forwarded-Proto`，排查 Nginx Proxy Manager、群晖反代、飞牛反代等场景时更容易看出外部访问地址是否被正确传递。
- **反代配置兼容性增强**：对只传 `Host` / `X-Forwarded-Host`、但没有传外部端口的常见 NAS / NPM 反代配置做兼容，降低公网访问部署门槛。

## 升级说明

- **所有部署形态**：本次不涉及数据库结构变更和配置变更，正常替换到 `v2.3.0` 即可。
- **Web 面板用户**：如果升级后仍看到旧的日志刷屏表现，请先强制刷新浏览器缓存，或重新打开面板页面，确保加载到新的前端静态资源。
- **Docker 用户**：请重新拉取 `v2.3.0` 镜像并重建容器；使用 Watchtower 的用户等待自动更新即可。
- **Nginx Proxy Manager / 群晖反代 / 飞牛反代用户**：建议继续传递 `Host`、`X-Forwarded-Host`、`X-Forwarded-Proto`，如果反代面板支持，也建议补充 `X-Forwarded-Port`，方便后续排查。
- **二进制 / Windows 单机版用户**：下载 `v2.3.0` 对应压缩包，替换可执行文件即可；请保留原数据目录。
- **Magisk / KernelSU / APatch 模块用户**：可通过模块管理器更新，也可以下载 `v2.3.0` 模块包手动刷入；数据目录会继续保留。
- **回退建议**：如果升级后遇到异常，建议先备份数据目录，再回退到上一稳定版本 `v2.2.20`。
