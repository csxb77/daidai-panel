## 修复

- **Web 面板进度条日志仍刷屏**：修复任务实时日志和执行日志详情页在 SSE 解析时丢失裸 `\r` 控制符的问题。现在进度条、下载状态等终端覆盖刷新输出会继续在同一行更新，不再因为每个流式分片都被当成新行而刷屏。
- **NAS + Nginx Proxy Manager 反代登录提示跨域**：修复群晖、飞牛等 NAS 容器通过公网域名和非标准端口反代访问时，反代只传域名却丢失外部端口，导致 `Origin=https://域名:端口` 被误判为跨域并拦截登录的问题。现在同一反代域名的外部端口会被正确兼容；如果反代明确传入 `X-Forwarded-Port`，仍会严格校验端口一致性。

## 优化

- **日志渲染更接近真实终端**：统一任务实时日志、执行日志详情、日志文件预览对裸 `\r`、`\n`、`\r\n` 的处理语义，减少不同入口看到的日志表现不一致。
- **CORS 拒绝日志更利于排查反代问题**：CORS 拒绝日志补充 `X-Forwarded-Port` 与 `X-Forwarded-Proto`，排查 Nginx Proxy Manager、群晖反代、飞牛反代等场景时更容易看出外部访问地址是否被正确传递。

## 升级说明

- **所有部署形态**：本次 Web 日志渲染修复不涉及数据库结构变更和配置变更；如果升级后 Web 面板仍看到旧表现，建议先强制刷新浏览器缓存或重新打开面板页面，确保加载到新的前端静态资源。
- **Docker 用户**：待补充是否需要重新拉取镜像、重建容器、检查挂载目录或 Watchtower 配置。使用 Nginx Proxy Manager / 群晖反代 / 飞牛反代访问时，建议让反代继续传递 `Host`、`X-Forwarded-Host`、`X-Forwarded-Proto`，如可配置也建议补充 `X-Forwarded-Port`。
- **二进制 / Windows 单机版用户**：待补充是否需要重新下载压缩包、替换可执行文件、保留数据目录。
- **Magisk / KernelSU / APatch 模块用户**：待补充模块升级注意事项、运行时检查方式、兼容性说明。
