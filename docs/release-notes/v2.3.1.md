# v2.3.1
<!-- release-title: 管理页显示与日志体验优化 -->

## 版本概览

`v2.3.1` 是 `v2.3.0` 之后的一次体验修复版本，主要收敛新一轮用户反馈的界面可用性、任务排序、浏览器焦点和日志阅读问题。

本次重点解决：

- 脚本目录较多时，新建文件 / 新建目录 / 上传脚本选择上级目录不方便。
- 禁用任务如果已经置顶，仍应该保留在置顶区域。
- 多个管理页顶部统计卡片重复占用空间，列表和日志显示区域偏小。
- 脚本运行器 / 调试窗口使用小弹窗，代码区和日志区显示内容太少。
- Edge / Chromium 场景下，面板窗口最小化后可能被延迟 `focus()` 拉回前台。
- 日志显示窗口滚动条太细、颜色太浅，有时用户看不清当前位置。

本次不涉及数据库结构变更，也不需要用户手动迁移数据。

## 修复

- **脚本管理目录太多时选择不方便**：优化脚本管理页的新建文件、新建目录、上传文件弹窗。上级目录 / 目标目录现在支持直接输入关键词搜索，目录较多时不用再一路向下滚动查找。
- **禁用的置顶定时任务被挤出置顶区**：修复定时任务列表默认排序中“任务状态”优先级高于“置顶状态”的问题。现在任务如果已经是置顶状态，即使被禁用，也会继续保持在置顶区域，不会被普通启用任务挤到后面。
- **定时任务页顶部重复统计占用空间**：移除定时任务页面顶部 4 个重复统计卡片，避免与首页统计重复，让任务列表更靠上显示，单屏可看到更多任务。
- **脚本调试窗口显示区域太小**：脚本管理页的代码运行器和调试运行界面改为全屏工作区，不再使用小弹窗模式，代码区和日志区可显示更多内容，方便调试和排错。
- **多个管理页顶部统计卡片占用列表空间**：同步移除订阅管理、环境变量、执行日志、依赖管理、通知渠道、用户管理、Open API 页面顶部的重复统计卡片，让表格、日志和操作区更靠上显示。
- **Edge 浏览器最小化后偶发自动恢复**：补充前端焦点保护。面板窗口已最小化、切到后台或页面不可见时，拦截弹窗、编辑器和第三方组件的延迟 `focus()` 调用，避免浏览器被程序化聚焦拉回前台。
- **日志窗口滚动条不明显**：增强任务实时日志、执行日志详情、依赖安装日志、订阅拉取日志、面板日志、脚本运行输出和调试日志等日志显示区域的滚动条可见性，减少滚动条过细、颜色太浅导致“看不见”的情况。

## 优化

- **目录选择交互更顺手**：脚本目录下拉框保留原有选择能力，同时补充搜索过滤、无匹配提示和空目录提示，方便在多层目录、多脚本仓库场景下快速定位目标目录。
- **定时任务排序更符合用户预期**：默认列表排序改为先尊重置顶状态，再按运行 / 启用 / 排队 / 禁用等状态分组，避免用户主动置顶的任务因状态变化而丢失展示优先级。
- **列表页面更干净**：减少不必要的重复信息展示，优先把空间留给表格、代码编辑器和日志这类真正需要操作的内容。
- **管理页可视区域更大**：订阅管理、环境变量、执行日志、依赖管理、通知渠道、用户管理、Open API 统一去掉顶部大卡片；其中依赖管理保留轻量 Tab 切换 Node.js / Python3 / Linux，避免删掉原有切换入口。
- **后台窗口不再抢焦点**：保留前台正常输入、弹窗自动聚焦、编辑器编辑体验；仅在页面已经失焦或隐藏时阻止自动聚焦，降低 Edge / Chromium 场景下“点最小化又弹回来”的概率。
- **日志阅读更清楚**：日志区域使用更宽、更高对比度的专用滚动条，并保留滚动条空间，方便用户判断当前日志是否还能继续向下或横向查看。

## 升级说明

- **所有部署形态**：本次不涉及数据库结构变更和配置变更，正常替换到 `v2.3.1` 即可。
- **Web 面板用户**：如果升级后仍看到旧界面布局或旧滚动条样式，请先强制刷新浏览器缓存，或重新打开面板页面，确保加载到新的前端静态资源。
- **Docker 用户**：请重新拉取 `v2.3.1` 镜像并重建容器；使用 Watchtower 的用户等待自动更新即可。
- **二进制 / Windows 单机版用户**：下载 `v2.3.1` 对应压缩包，替换可执行文件即可；请保留原数据目录。
- **Magisk / KernelSU / APatch 模块用户**：可通过模块管理器更新，也可以下载 `v2.3.1` 模块包手动刷入；数据目录会继续保留。
- **回退建议**：如果升级后遇到异常，建议先备份数据目录，再回退到上一稳定版本 `v2.3.0`。
