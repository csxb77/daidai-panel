# v2.3.2
<!-- release-title: 定时任务快捷排序与订阅白名单说明 -->

## 版本概览

`v2.3.2` 是 `v2.3.1` 之后的一次功能增强 + 体验优化版本，主要给定时任务加上快捷排序，并澄清订阅白名单会参与实际文件检出范围的行为。

本次重点：

- 定时任务列表新增快捷排序：可按名称、创建时间排序，或恢复默认排序。
- 订阅白名单行为说明：白名单不仅筛选任务，还会参与 sparse checkout 的实际检出范围；主脚本依赖的辅助文件需一并加入白名单。

本次不涉及数据库结构变更，也不需要用户手动迁移数据。

## 优化

- **定时任务快捷排序**：工具栏新增「排序」下拉，可按名称（A→Z / Z→A）或创建时间（最新 / 最早优先）快速排序，或选「默认排序」恢复原有的「置顶 + 状态分组」排序。分页场景下排序由后端处理，覆盖全部任务而非仅当前页。
- **订阅白名单行为说明**：澄清白名单不仅用于筛选任务，还会参与 sparse checkout 的实际检出范围——只有命中白名单的文件会落盘。在订阅表单白名单字段下新增提示，并标注「依赖说明仅备注、不参与检出」，README 同步说明，提醒把主脚本依赖的辅助文件一并加入白名单，避免升级后出现 Cannot find module / 文件不存在。

## 升级说明

- **所有部署形态**：本次不涉及数据库结构变更和配置变更，正常替换到 `v2.3.2` 即可。
- **Web 面板用户**：如果升级后仍看到旧界面布局或旧样式，请先强制刷新浏览器缓存（Ctrl+Shift+R），确保加载到新的前端静态资源。
- **Docker 用户**：请重新拉取 `v2.3.2` 镜像并重建容器；使用 Watchtower 的用户等待自动更新即可。
- **二进制 / Windows 单机版用户**：下载 `v2.3.2` 对应压缩包，替换可执行文件即可；请保留原数据目录。
- **Magisk / KernelSU / APatch 模块用户**：可通过模块管理器更新，也可以下载 `v2.3.2` 模块包手动刷入；数据目录会继续保留。
- **回退建议**：如果升级后遇到异常，建议先备份数据目录，再回退到上一稳定版本 `v2.3.1`。
