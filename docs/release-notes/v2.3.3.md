# v2.3.3
<!-- release-title: 依赖去重 · 通知增强 · 任务标签与日志清理 -->

## 版本概览

`v2.3.3` 是 `v2.3.2` 之后的一次功能增强 + 体验优化 + 问题修复版本，集中处理一批用户反馈的实际问题：依赖重复、通知可读性与来源区分、任务标签管理、日志清理、手动停止误判，以及若干界面细节。

本次重点：

- Python 依赖按 PyPI 规则（PEP 503）归一化去重，`Requests` 与 `requests` 视为同一个包，升级时自动合并已有重复记录。
- 钉钉机器人新增「文本 / Markdown」消息类型；通知可附带面板名称前缀，多面板共用渠道时一眼区分来源。
- 定时任务支持批量添加标签，配合按标签的视图归类任务。
- 修复日志保留天数从未自动生效、手动停止任务被误判为失败、收起态侧栏图标未对齐等问题。

本次不涉及数据库结构变更，也不需要用户手动迁移数据（依赖重复记录会在升级启动时自动合并）。

## 新增

- **钉钉消息类型可配置**：钉钉机器人渠道新增「消息类型」选项，可选「文本」或「Markdown」。此前钉钉固定走 Markdown，渲染会折叠多余空格、打乱对齐文本，导致脚本输出里的符号 / emoji / 对齐表格显示异常；选「文本」后内容原样发出。老渠道未设置时仍按 Markdown 发送，行为不变。
- **通知附带面板名称**：系统设置新增「通知面板名称」配置项（告警相关配置内）。填写后所有通知渠道的标题统一加上 `【面板名】` 前缀，方便多个面板共用同一通知渠道时区分来源；留空则不附带，存量用户无感。
- **定时任务批量添加标签**：定时任务列表选中多个任务后，批量操作栏新增「添加标签」，可一次给多个任务追加标签（保留原有标签、去重）。配合「按标签包含」的自定义视图，即可把任务按标签归类查看。

## 修复

- **日志保留天数从未自动清理**：此前「日志保留天数（默认 7 天）」只在手动触发清理时才生效，没有任何定时任务执行它，导致日志实际从不自动删除。现在新增后台清理任务，启动后先清一次、之后每 6 小时一次，按保留天数同时清理数据库执行记录与磁盘日志文件。
- **收起态侧栏图标未对齐**：修复左侧导航收起后，导航图标列与顶部 logo 图标不在同一竖直轴、整体看起来歪斜的问题，现已统一居中对齐。
- **日志窗口"运行中却显示已完成"**：修复任务仍在运行，但实时日志窗口因连续 60 秒无新输出（如 sleep、等慢接口、长计算、下载），或在任务刚启动的瞬间，就误显示"已完成"的问题。现在以任务真实运行状态为准：安静运行的长任务会保持"运行中"并自动续流，任务真正结束后才显示"已完成"。

## 优化

- **Python 依赖大小写重复去重**：pip / PyPI 本身按 PEP 503 把包名统一规范化（转小写、`-_.` 折叠成 `-`），`Requests` 与 `requests`、`Flask_SQLAlchemy` 与 `flask-sqlalchemy` 都是同一个包。此前面板入库不归一化也不查重，导致依赖列表出现同名重复行，卸载其一会连带使另一条失效。现在手动安装与脚本自动安装两条入口都按归一化键去重：已存在的同名包（含带版本号 / extras 的写法）直接跳过、不重复安装，并提示「已存在跳过 N 个」；升级启动时还会一次性合并已有的大小写 / 分隔符重复行，保留已安装的那条。本次去重仅作用于 Python 依赖，npm / Linux 依赖不受影响。
- **手动停止任务不再判为失败**：手动停止正在运行的任务，运行结果由「失败」改判为「成功」，且不再发送失败 / 成功通知，避免把人为主动停止误报为执行失败。真实执行失败仍判失败、仍发失败通知。
- **环境变量名即时校验提示**：新建 / 编辑环境变量时，变量名输入框新增规则提示「只能输入字母、数字、下划线，且不能以数字开头」，并在不符合时即时标红、阻止提交，避免提交后才被后端拒绝。

## 升级说明

- **所有部署形态**：本次不涉及数据库结构变更和配置变更，正常替换到 `v2.3.3` 即可；已有的 Python 依赖大小写重复行会在升级首次启动时自动合并。
- **Web 面板用户**：如果升级后仍看到旧界面布局或旧样式，请先强制刷新浏览器缓存（Ctrl+Shift+R），确保加载到新的前端静态资源。
- **钉钉通知用户**：如需切换为文本消息，请到「通知渠道」编辑钉钉渠道、把「消息类型」改为「文本」并保存；不改则维持原有 Markdown 行为。
- **多面板用户**：如需在通知里区分来源，请到「系统设置」填写「通知面板名称」。
- **Docker 用户**：请重新拉取 `v2.3.3` 镜像并重建容器；使用 Watchtower 的用户等待自动更新即可。
- **二进制 / Windows 单机版用户**：下载 `v2.3.3` 对应压缩包，替换可执行文件即可；请保留原数据目录。
- **Magisk / KernelSU / APatch 模块用户**：可通过模块管理器更新，也可以下载 `v2.3.3` 模块包手动刷入；数据目录会继续保留。
- **回退建议**：如果升级后遇到异常，建议先备份数据目录，再回退到上一稳定版本 `v2.3.2`。
