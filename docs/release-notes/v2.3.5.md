# v2.3.5
<!-- release-title: 任务终止状态、开机触发与 Python 镜像拆分 -->

## 版本概览

`v2.3.5` 重点解决任务执行状态、备份恢复和 Docker Python 运行时体积问题。

本次更新包含三类核心改动：

- 修复「开机运行」任务在面板更新、面板重启或电脑重启后当天重复自动执行的问题：同一个开机运行任务现在每天只会被面板启动流程自动触发一次，手动点击「运行」仍然可以多次触发。
- 增加完整的任务终止状态 `Aborted`：手动停止、批量停止、定时停止和 CLI 停止会统一标记为「已终止」，不再混入成功或失败统计，也不会误发失败通知。
- Docker 镜像按 Python 小版本拆分：默认 `latest` / `debian` 只保留 Python `3.12`，新增 `latest3.10`、`latest3.11`、`debian3.10`、`debian3.11`，以及三版本合集 `latestall`、`debianall`。

本次升级不需要手动迁移数据库。面板启动时会自动补齐新增字段，并在单 Python 镜像中按当前镜像策略清理不再支持的托管 Python 环境目录。

## 新增

- **任务终止状态 Aborted**：任务被手动停止、批量停止、定时停止或通过 `ddp stop` 停止时，现在会进入独立的「已终止」状态。该状态会写入任务最后运行状态和任务日志，前端任务列表、任务详情、日志列表、仪表盘和执行趋势都会单独展示「已终止」，不再把计划内终止伪装成成功或失败。
- **终止通知开关**：任务高级设置新增「终止时通知」开关，对应字段为 `notify_on_abort`。默认关闭；开启后，任务被主动终止时会发送「任务已终止」通知。该通知独立于成功通知和失败通知，避免把终止误报成失败。
- **Python 版本拆分 Docker 镜像**：新增单 Python 小版本镜像 `linzixuanzz/daidai-panel:latest3.10`、`linzixuanzz/daidai-panel:latest3.11`、`linzixuanzz/daidai-panel:debian3.10`、`linzixuanzz/daidai-panel:debian3.11`；新增三版本合集镜像 `linzixuanzz/daidai-panel:latestall`、`linzixuanzz/daidai-panel:debianall`。默认 `latest` 和 `debian` 现在只内置 Python `3.12`。
- **镜像架构更清晰**：默认 Alpine `latest` 继续保留 `amd64 / arm64 / 386 / armv7`；Alpine 的 `latest3.10`、`latest3.11`、`latestall` 只发布 `amd64 / arm64`，避免在没有独立 Python 资产的 32 位平台上生成“标签是 3.10，实际不是 3.10”的假镜像。Debian 变体继续发布 `amd64 / arm64 / armv7`。

## 修复

- **开机运行任务当天重复自动执行**：此前任务设置为「开机运行」后，只要面板服务启动就会自动入队执行一次；因此面板内更新重建容器、手动重启面板、电脑重启后重新启动面板，都会在同一天再次触发这些开机运行任务。现在为任务新增「开机运行自动触发日期」记录，面板启动时只会自动入队当天尚未自动触发过的开机运行任务；任务执行结束后即使状态回到「启用」，当天再次重启也不会重复自动执行。手动点击「运行」不受该限制，仍可按需多次执行。
- **恢复备份后环境变量禁用状态丢失**：修复环境变量恢复时 `enabled=false` 被 GORM 默认值覆盖成 `true` 的问题。现在备份会明确记录环境变量启用状态，恢复时会按备份内容还原；禁用的环境变量不会再被恢复成启用。旧备份如果缺少 `enabled` 字段，仍按历史行为默认启用，保证兼容。
- **定时停止任务误判失败**：主动停止、批量停止、定时停止和 CLI 停止现在都会先写入停止标记，再终止进程；任务结算时会统一改写为 `Aborted`，避免进程被杀后因为非 0 退出码被误判为运行失败。
- **停止任务误发失败通知**：`Aborted` 不再触发成功通知或失败通知；只有任务开启「终止时通知」时，才会发送独立的终止通知。
- **停止任务污染失败统计**：仪表盘和统计接口新增终止统计，`Aborted` 不再增加成功或失败计数，成功率仅按 `成功 / (成功 + 失败)` 计算。

## 优化

- **单 Python 镜像自动收敛历史配置**：当用户从旧版三 Python 镜像升级到新版默认 `latest` / `debian` 后，面板启动时会把系统默认 Python 版本和历史任务 `python_version` 自动切回当前镜像版本，默认即 `3.12`。
- **自动清理不支持的托管 Python 环境**：单版本镜像启动时只会保留当前镜像支持的小版本目录。例如默认 `latest` 会保留 `data/deps/python/3.12`，并清理 `data/deps/python/3.10`、`data/deps/python/3.11`；`latestall` / `debianall` 不会清理三版本目录。
- **依赖管理按当前镜像显示 Python 版本**：依赖管理页和任务表单现在会根据当前 Docker 镜像支持的 Python 版本动态展示选项。单版本镜像只显示当前小版本，避免用户继续选择镜像中不存在的 Python。
- **不再静默 fallback 到错误版本**：当前镜像不支持用户选择的 Python 小版本时，后端会明确返回错误，提示切换到对应 Python 版本镜像或 `all` 镜像，避免出现“选择 3.10，实际使用 3.12”的隐性问题。
- **发布预检增强**：发版预检脚本新增 Docker Python 镜像 tag、matrix 平台列表和 Dockerfile 构建参数检查，减少发版时漏配镜像变体的风险。

## Docker 镜像说明

从 `v2.3.5` 起，Docker 镜像按 Python 小版本拆分：

| 镜像 tag | Python 运行时 | 平台 |
|---|---|---|
| `latest` / `<版本号>` | 单版本 `3.12` | Alpine：`amd64 / arm64 / 386 / arm/v7` |
| `latest3.10` / `<版本号>-3.10` | 单版本 `3.10` | Alpine：`amd64 / arm64` |
| `latest3.11` / `<版本号>-3.11` | 单版本 `3.11` | Alpine：`amd64 / arm64` |
| `latestall` / `<版本号>-all` | `3.10 / 3.11 / 3.12` | Alpine：`amd64 / arm64` |
| `debian` / `<版本号>-debian` | 单版本 `3.12` | Debian：`amd64 / arm64 / arm/v7` |
| `debian3.10` / `<版本号>-debian3.10` | 单版本 `3.10` | Debian：`amd64 / arm64 / arm/v7` |
| `debian3.11` / `<版本号>-debian3.11` | 单版本 `3.11` | Debian：`amd64 / arm64 / arm/v7` |
| `debianall` / `<版本号>-debianall` | `3.10 / 3.11 / 3.12` | Debian：`amd64 / arm64 / arm/v7` |

如果你一直使用默认 `latest` 或 `debian`，升级后会默认使用 Python `3.12`。如果已有任务强依赖 Python `3.10` 或 `3.11`，请把 Docker tag 改为对应小版本镜像，或改用 `latestall` / `debianall`。

## 数据库兼容

本次更新会自动补齐以下字段：

- `tasks.last_startup_auto_run_date`：记录开机运行任务当天是否已经由面板启动流程自动触发。
- `tasks.notify_on_abort`：控制任务被主动终止后是否发送终止通知。

升级时会自动补列，不需要用户手动执行 SQL。
