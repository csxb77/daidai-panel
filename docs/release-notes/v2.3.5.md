## 新增

- **任务终止状态 Aborted**：任务被手动停止、批量停止、定时停止或通过 `ddp stop` 停止时，现在会进入独立的「已终止」状态。该状态会写入任务最后运行状态和任务日志，前端任务列表、任务详情、日志列表、仪表盘和执行趋势都会单独展示「已终止」，不再把计划内终止伪装成成功或失败。
- **终止通知开关**：任务高级设置新增「终止时通知」开关，对应字段为 `notify_on_abort`。默认关闭；开启后，任务被主动终止时会发送「任务已终止」通知。该通知独立于成功通知和失败通知，避免把终止误报成失败。

## 修复

- **开机运行任务当天重复自动执行**：此前任务设置为「开机运行」后，只要面板服务启动就会自动入队执行一次；因此面板内更新重建容器、手动重启面板、电脑重启后重新启动面板，都会在同一天再次触发这些开机运行任务。现在为任务新增「开机运行自动触发日期」记录，面板启动时只会自动入队当天尚未自动触发过的开机运行任务；任务执行结束后即使状态回到「启用」，当天再次重启也不会重复自动执行。手动点击「运行」不受该限制，仍可按需多次执行。

## 优化

- **定时停止 / 主动停止统计口径**：此前任务被定时停止或手动停止后容易被当成运行失败，导致失败通知、失败统计和任务成功率都不符合「计划内停止」的直觉。现在主动终止会进入 `Aborted` 独立状态，统计中新增终止数量，成功率只按自然完成的「成功 / 失败」计算，终止不会拉低成功率，也不会增加失败记录。
- **终止状态前端展示**：任务列表和任务详情的上次运行状态会显示「已终止」；执行日志支持按「已终止」筛选；仪表盘新增终止日志统计，执行趋势图也会展示终止曲线，方便区分脚本异常失败和用户主动终止。
