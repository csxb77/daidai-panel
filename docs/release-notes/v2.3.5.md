- 优化「定时停止 / 手动停止」后的任务结算：默认把主动停止视为成功，避免误发失败通知、误增失败统计；同时新增任务级「终止算失败」开关，给需要把终止计为失败的用户保留选择。

## 修复

- **开机运行任务当天重复自动执行**：此前任务设置为「开机运行」后，只要面板服务启动就会自动入队执行一次；因此面板内更新重建容器、手动重启面板、电脑重启后重新启动面板，都会在同一天再次触发这些开机运行任务。现在为任务新增「开机运行自动触发日期」记录，面板启动时只会自动入队当天尚未自动触发过的开机运行任务；任务执行结束后即使状态回到「启用」，当天再次重启也不会重复自动执行。手动点击「运行」不受该限制，仍可按需多次执行。

## 优化

- **定时停止任务结算口径**：此前任务被定时停止后容易被当成运行失败，导致失败通知、失败统计和任务成功率都不符合「计划内停止」的直觉。现在主动停止任务默认按成功结算，并跳过成功 / 失败任务通知；如果确实希望把终止计为失败，可在任务高级设置里开启「终止算失败」，开启后失败通知和失败统计会继续按失败处理。

## 升级说明

- **所有部署形态**：本次已新增 `tasks.last_startup_auto_run_date` 和 `tasks.stop_as_failure` 兼容字段，分别用于限制开机运行任务每天只自动触发一次、控制主动停止后的成功 / 失败结算；升级启动时会自动补列，不需要用户手动迁移数据。正常替换到 `v2.3.5` 即可。
- **Web 面板用户**：如果本次涉及前端界面、样式或交互改动，升级后如仍看到旧界面，请先强制刷新浏览器缓存（Ctrl+Shift+R），确保加载到新的前端静态资源。
