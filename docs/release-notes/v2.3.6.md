# v2.3.6
<!-- release-title: Linux 二进制发行包时区修复 -->

## 版本概览

`v2.3.6` 先记录 Linux 二进制发行包的时区修复。

此前 `daidai-linux-arm64.tar.gz` 这类裸二进制包如果运行在默认 UTC、缺少 `TZ` 或 `/etc/localtime` 不正确的精简系统里，面板日志、任务日志和脚本运行时间可能会比 Web 面板看到的本地时间少 8 小时。

本次修复后，面板会提供统一的“面板时区”配置，默认使用 `Asia/Shanghai`，并在启动和保存配置时立即应用到 Go 进程与任务脚本环境。

## 修复

- **修复 Linux 二进制包默认 UTC 导致日志时间相差 8 小时**：新增系统配置项 `timezone`，默认值为 `Asia/Shanghai`。面板启动后会把该时区应用到 `time.Local` 和进程环境变量 `TZ`，后续面板访问日志、任务日志、定时任务日期判断都会使用同一个面板时区。
- **修复任务脚本继承不到正确时区**：任务运行环境会自动注入当前面板时区对应的 `TZ`，并覆盖普通环境变量里的 `TZ`，避免脚本继续按 UTC 计算时间。
- **修复精简 Linux 缺少系统 tzdata 时无法加载时区的问题**：后端内嵌 Go 官方时区数据，`Asia/Shanghai`、`UTC`、`Asia/Tokyo` 等 IANA 时区名不再依赖宿主机必须安装完整 tzdata。

## 新增

- **系统设置页新增“面板时区”**：用户可以在“面板设置”中选择或输入 IANA 时区名，例如 `Asia/Shanghai`、`UTC`、`Asia/Tokyo`。
- **无效时区校验**：保存配置时会校验时区名是否有效；无效值会被拒绝，避免保存后任务时间继续错乱。
- **新增独立“配置文件”页面**：将原本藏在“环境变量”更多菜单里的 `config.sh` 移到左侧导航，管理员可以直接进入页面查看、复制、刷新和保存配置文件。

## 升级说明

- 新安装或旧版本升级后，如果没有保存过 `timezone` 配置，默认使用 `Asia/Shanghai`。
- Docker 场景原本通过 `TZ=Asia/Shanghai` 启动的用户不受影响；如需其它地区时区，可以在系统设置里修改“面板时区”。
- 本次不会修改历史日志中已经写入的旧时间，只保证升级后新产生的日志和任务运行环境使用正确时区。

## 验证

- 后端已增加回归测试，覆盖默认时区、无效时区、保存后立即生效、任务环境注入 `TZ`。
- 前端构建与全量后端测试结果以最终发布前验证为准。

## 本轮追加修复

- **修复部分青龙 Node 脚本在呆呆里只输出“开始”就静默结束的问题**：兼容 `hex-ci/smzdm_script` 这类脚本里 `JSON.stringify(process.env).indexOf("GITHUB")` 的粗暴 CI 检测。呆呆现在只会在 `JSON.stringify(process.env)` 的结果里隐藏包含大写 `GITHUB` 的环境项，不删除真实 `process.env.GITHUB_*`，所以脚本不会误判退出，同时显式读取 GitHub 相关变量的脚本也不受影响。
- **移除环境变量页里的隐藏配置文件入口**：环境变量页面的更多菜单只保留导入导出能力，`config.sh` 统一走左侧“配置文件”页面，避免用户找不到入口。
